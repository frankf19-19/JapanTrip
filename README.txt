【部署方式(GitHub Pages,零成本)】
1. 到 GitHub 建立新 repo(例:frankf19-19/JapanTrip)
2. 上傳 index.html 到 repo 根目錄
3. Settings → Pages → Source 選 main branch / root → Save
4. 網址:https://frankf19-19.github.io/JapanTrip/

【功能總覽】
● 定位:輸入任意日本地址/地標(Nominatim 免金鑰地理編碼)或快選 9 大城市
● 喜好篩選:神社寺廟/歷史古蹟/自然/市場老街/主題樂園/動漫/博物館/夜景/溫泉/親子
● 天數 1–7 天 + 節奏(輕鬆/標準/緊湊)+ 搜尋半徑 5–80km
● 自動生成行程:就近分群 + 鐵道路線圖時間軸 + 自動插入午/晚餐 + 住宿建議
● 探索頁:126 筆精選(全含照片・地址・價格・評分)(景點/美食/購物/住宿)+ OSM Overpass 即時搜尋補充
● 地圖頁:Leaflet 顯示每日路線(不同顏色)
● 匯出:一鍵複製行程文字 / 列印存 PDF
● 手機 App 式底部導覽,桌機頂部導覽

【資料來源與限制】
- 精選資料庫為內建靜態資料,座標與描述僅供參考
- 即時搜尋使用 Overpass 公共伺服器,尖峰時段可能逾時
- 大型樂園(迪士尼/USJ)自動獨佔一整天

【啟用擴充庫步驟】
1. 上傳整包(index.html + scripts/ + .github/)到 repo
2. repo Settings → Actions → General → Workflow permissions
   選「Read and write permissions」→ Save
3. Actions 頁 → 「建置和遊誌擴充景點資料庫」→ Run workflow
4. 跑完(約 5–10 分)repo 會多出 data/spots_extra.json,
   網站重新整理即會顯示「📚 全國擴充資料庫已載入 +N 筆」
